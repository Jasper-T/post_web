# 操作指南

本文档面向前端页面使用者，介绍如何使用本工具完成图片算法接口调试、批量测试、结果查看、可视化保存、结果评估和数据集辅助处理。

## 1. Windows 便携版

### 1.1 发布后的目录结构

Windows 便携版发布后，目录结构如下：

```text
fuxing/
  fuxing.exe
  start.bat
  fuxing.env
  html/
  data/
  datasets/
```

不要单独移动 `fuxing.exe`。请保持 `fuxing.exe`、`html`、`data`、`datasets` 在同一个 `fuxing` 文件夹里。

### 1.2 发布包目录说明

- `html/`：前端页面文件，不需要手动修改。
- `data/`：保存 Pipeline、分组、日志和运行结果。
- `datasets/`：上传按钮的固定目标目录，也就是后端机器接收数据的位置。

文件浏览和上传的范围是故意分开的：

- 文件浏览弹窗显示的是后端机器上可见的目录，根目录由 `FUXING_FILESYSTEM_ROOT` 控制。
- Windows 便携版默认显示 `fuxing.exe` 所在磁盘，例如程序在 `D:\tools\fuxing`，默认根目录就是 `D:\`。
- 上传按钮选择文件时，浏览器会让你从前端电脑上任意位置选择文件。
- 上传目标固定是后端机器的 `FUXING_DATASETS_ROOT`，便携版默认是 `fuxing/datasets`。

这样设计是为了支持“前端在 A 机器打开，后端在 B 机器运行”的情况：A 机器选择文件，文件会上传到 B 机器的 `datasets` 目录。

### 1.3 启动方法

1. 打开 `fuxing` 文件夹。
2. 双击 `fuxing.exe` 或 `start.bat`。
3. 等几秒钟，浏览器通常会自动打开。
4. 如果浏览器没有自动打开，手动访问：`http://127.0.0.1:8000`。

如果 8000 端口被占用，可以用命令行指定其他端口。

使用 CMD 启动：

1. 打开 `fuxing` 文件夹。
2. 点击资源管理器顶部地址栏。
3. 输入 `cmd`。
4. 按回车。
5. 输入下面的命令：

```bat
fuxing.exe --port 8001
```

使用 PowerShell 启动：

1. 打开 `fuxing` 文件夹。
2. 按住 `Shift`，在文件夹空白处右键。
3. 点击“在终端中打开”或“在此处打开 PowerShell”。
4. 输入下面的命令：

```powershell
.\fuxing.exe --port 8001
```

然后访问：`http://127.0.0.1:8001`。

## 2. 打开页面

1. 启动程序后，在浏览器访问页面地址。
   - Windows 便携版默认地址：`http://127.0.0.1:8000`
   - 如果启动时指定了其他端口，请访问对应端口，例如：`http://127.0.0.1:8001`
2. 页面左侧是 Pipeline 列表，中间是请求配置和响应展示工作区。
3. 第一次使用时，可以先选择默认的 `Demo Image Pipeline` 进行测试。

## 3. 页面区域说明

### 3.1 左侧 Pipeline 列表

左侧用于管理接口调试配置，也就是 Pipeline。

- 点击分组名称可以展开或收起分组。
- 点击某个 Pipeline 可以加载它的配置。
- 点击左上方的 `Edit` 可以打开 Collections Editor，用于新建分组、重命名分组、移动 Pipeline、删除 Pipeline。
- 在分组或 Pipeline 上右键，可以快速新建、复制、移动或删除。
- `Deleted` 分组用于存放已删除的 Pipeline；在该分组中再次删除会永久删除。

### 3.2 顶部请求栏

顶部用于配置接口基本信息。

- 请求方法：目前支持`POST`方法。
- URL：填写算法接口地址，例如 `/api/mock/detection` 或完整接口地址。
- `new`：新建一个空 Pipeline。
- `clone`：复制当前 Pipeline。
- 保存按钮：保存当前 Pipeline 的基本信息。

### 3.3 Pipeline 基本信息

请求栏下方包含：

- `Pipeline Name`：Pipeline 的唯一名称。使用小写字母、数字、下划线或短横线。
- `Display Name`：页面显示名称，可以写得更易读。
- `Image Path`：待测试图片或图片目录路径。
- `Browse`：打开文件浏览器选择图片或目录。

## 4. 创建或编辑 Pipeline

### 4.1 新建 Pipeline

1. 点击顶部 `new`，或在左侧分组右键选择 `New pipeline`。
2. 填写 `Pipeline Name`、`Display Name`、请求方法和 URL。
3. 点击保存按钮保存。

`Pipeline Name` 需要满足以下规则：

- 长度为 3-64 个字符。
- 必须以小写字母开头。
- 只能包含小写字母、数字、`_` 和 `-`。
- 不能以 `_` 或 `-` 结尾。
- 不能包含连续分隔符，例如 `__`、`--`、`_-`、`-_`。

### 4.2 配置请求内容

点击工作区中的 `请求解析`，依次配置以下标签页：

1. `Header`
   - 配置请求头 JSON。
   - 点击 `Edit` 打开编辑器。
   - 可使用 `JSON` 模式直接粘贴 JSON，也可切换到 `Tree` 模式按字段编辑。
   - 点击 `Save` 保存。

2. `Body`
   - 配置请求体 JSON。
   - 如果接口需要图片 base64，请在请求体中预留一个字段，例如 `image`、`image_b64` 或业务接口要求的字段。
   - 保存后，系统会基于 Body 字段生成后续请求配置可选路径。

3. `Response`
   - 填写接口返回示例 JSON。
   - 建议使用真实接口返回的一条样例。
   - 保存后，系统会基于 Response 字段生成响应映射可选路径。

4. `响应映射`
   - 用于告诉系统如何从接口响应中提取检测框、类别、置信度等信息。
   - 需要先保存 `Body` 和 `Response`，该标签页才可编辑。

5. `请求配置`
   - 用于设置超时时间和动态字段注入位置。
   - 需要先保存 `Body` 和 `Response`，该标签页才可编辑。

## 5. 配置响应映射

进入 `响应映射` 标签页后，按实际响应结构选择字段路径。

常用字段含义：

- `Collection Path`：响应中检测结果列表所在路径。
- `Item Path`：如果每个 Collection 内还有一层列表，在这里选择子列表路径。
- `bbox` 或 `x1/y1/x2/y2`：检测框坐标。
- `label`：类别名称。
- `class_id`：类别 ID。
- `conf`：置信度。
- `Extra Params`：额外字段，例如 track_id、属性值、文本等。
- `Plot`：勾选后，该字段会显示在可视化标注文字中。

坐标模式：

- `BBox`：响应中已有一个数组形式的检测框，例如 `[x1, y1, x2, y2]`。
- `Point`：响应中坐标分散在多个字段中，例如 `x1`、`y1`、`x2`、`y2`。
- `XYXY`：左上角和右下角坐标。
- `XYWH`：左上角坐标加宽高。

配置完成后点击 `Save`。

## 6. 配置请求参数注入

进入 `请求配置` 标签页。

可以设置：

- `Connect Timeout (s)`：连接超时时间。
- `Read Timeout (s)`：读取响应超时时间。
- `timestamp`：当前时间戳注入到 Body 的哪个字段。
- `image_b64`：图片 base64 注入到 Body 的哪个字段。
- `image_width`：图片宽度注入到 Body 的哪个字段。
- `image_height`：图片高度注入到 Body 的哪个字段。

如果接口需要接收图片，通常必须设置 `image_b64` 的注入路径。

配置完成后点击 `Save`。

## 7. 选择或上传图片

### 7.1 选择后端已有图片

1. 在 `Image Path` 右侧点击 `Browse`。
2. 在文件浏览器中展开目录。
3. 选择单张图片或图片目录。
4. 关闭文件浏览器后，页面会加载图片列表。

文件浏览器中的目录是后端机器可访问的目录，不一定等同于当前浏览器所在电脑的目录。

### 7.2 上传本机文件或文件夹

1. 打开 `Browse` 文件浏览器。
2. 在允许上传的位置选择目标目录。
3. 点击上传按钮。
4. 可以拖拽文件或文件夹到上传区域，也可以点击选择文件或文件夹。
5. 等待进度完成。
6. 上传完成后，刷新或重新选择目录即可看到新文件。

上传目标通常是后端的 `datasets` 目录。

## 8. 运行接口测试

1. 选择一个 Pipeline。
2. 确认请求方法、URL、Header、Body、Response、响应映射和请求配置已保存。
3. 在 `Image Path` 选择图片或图片目录。
4. 切换到 `响应展示`。
5. 右侧图片列表会显示当前图片队列。
6. 勾选要测试的图片。
7. 点击 `Send All` 批量发送。
8. 如只测试单张图片，可点击图片行上的发送按钮。

图片状态说明：

- `new`：尚未发送。
- `success`：请求成功并解析完成。
- `failed`：请求失败或解析失败。

右上角可用状态筛选器查看 `All`、`New`、`Success`、`Failed`。

## 9. 查看结果

在 `响应展示` 中可切换以下标签页：

### 9.1 原始响应

显示接口返回的原始 JSON。适合检查接口是否正常返回、字段名是否符合预期。

### 9.2 解析结果

显示经过响应映射后的标准结果。适合检查 bbox、label、class_id、conf 等字段是否解析正确。

### 9.3 图片预览

显示图片及检测框可视化。

- `Pred`：查看接口预测结果。
- `GT`：查看真实标注结果。
- 在图片预览区域获得焦点后，可使用键盘左右方向键切换图片。
- 点击保存按钮可以保存当前可视化图片。

使用 `GT` 时，需要选择标注格式和标注文件夹：

- 支持 `YOLO`、`LabelMe`、`VOC`。
- 类别名称可通过类别名称按钮维护。
- 标注文件夹应与所选格式匹配。

### 9.4 标注转换

显示可用于标注转换的解析结果。通常在响应映射正确后使用。

### 9.5 结果评估

`结果评估` 用于把当前 Pipeline 的预测结果与真实标注进行对比，输出 Precision、Recall、F1、mAP、TP、FP、FN 等指标。

使用前请先准备好：

- 当前图片已经运行成功，并且 `解析结果` 中能看到预测框。
- 已经选择图片或图片目录。
- 已经填写类别名称。类别顺序需要和真实标注中的类别 ID 对应。
- 已经选择 GT 标注格式和 GT 标注文件夹。

操作步骤：

1. 在 `响应展示` 中打开 `结果评估`。
2. 点击 `类别名称`，维护类别列表。
3. 选择 GT 标注格式，支持 `YOLO`、`LabelMe`、`VOC`。
4. 点击标注文件夹按钮，选择真实标注所在目录。
5. 按需要调整 `Conf` 和 `IoU` 阈值。
6. 点击 `开始评估`。
7. 查看整体指标和按类别统计表。

指标含义：

- `Precision`：预测为目标的结果中，有多少是真的目标。
- `Recall`：真实目标中，有多少被检测出来。
- `F1`：Precision 和 Recall 的综合指标。
- `mAP`：综合衡量检测效果的平均精度指标。
- `TP`：正确检测数量。
- `FP`：误检数量。
- `FN`：漏检数量。
- `Images`：参与评估的图片数量。

如果 `开始评估` 按钮不可点击，请检查是否已经运行图片、填写类别名称并选择 GT 标注文件夹。

## 10. 加载历史结果

如果之前运行过 Pipeline，可以在 `响应展示` 右侧点击 `加载结果`。

1. 点击 `加载结果`。
2. 选择结果文件夹。
3. 页面会把历史结果加载回图片列表。
4. 可继续查看原始响应、解析结果和图片预览。

## 11. 保存可视化图片

1. 在 `响应展示` 中打开 `图片预览`。
2. 选择 `Pred` 或 `GT`。
3. 确认当前图片已经有可视化结果。
4. 点击保存按钮。
5. 在弹窗中确认保存目录和文件名。
6. 保存完成后，可在对应目录查看输出图片。

## 12. 使用 dsetkit 工具

页面提供 dsetkit 工具，用于数据集辅助处理。

常用功能：

- 标注格式转换：在 `LabelMe`、`VOC`、`YOLO` 之间转换标注格式。
- 标注可视化：把标注框绘制到图片上并批量导出。
- 结果评估：在 `响应展示` 的 `结果评估` 页签中，对当前预测结果和 GT 标注进行指标评估。

使用步骤：

1. 在文件树中选择当前工作目录。
2. 选择工具类型。
3. 填写图片目录。
4. 填写标签目录。
5. 选择源格式。
6. 如果是格式转换，还需要选择目标格式。
7. 填写类别名称，每行一个类别，或用英文逗号分隔。
8. 可选填写输出目录；不填时使用默认输出位置。
9. 点击执行工具。

## 13. 推荐的完整工作流程

第一次配置新接口时，建议按以下顺序操作：

1. 新建 Pipeline。
2. 填写请求方法和 URL。
3. 配置并保存 `Header`。
4. 配置并保存 `Body`，确认图片 base64 字段存在。
5. 粘贴真实返回样例并保存 `Response`。
6. 配置并保存 `响应映射`。
7. 配置并保存 `请求配置`，重点设置 `image_b64` 注入路径。
8. 选择一张图片，单张发送测试。
9. 查看 `原始响应` 和 `解析结果`。
10. 查看 `图片预览`，确认检测框位置正确。
11. 批量勾选图片，点击 `Send All`。
12. 如需量化效果，进入 `结果评估`，选择 GT 标注并点击 `开始评估`。
13. 保存或加载结果，按需要导出可视化图片。

## 14. 常见问题

### 14.1 响应映射或请求配置无法点击

请先保存 `Body` 和 `Response`。这两个 JSON 保存后，系统才能生成可选字段路径。

### 14.2 点击 Send All 没有反应

请检查：

- Pipeline 是否已保存。
- `Image Path` 是否已选择。
- 右侧图片列表中是否勾选了图片。
- URL、Body、请求配置是否完整。

### 14.3 请求失败

请检查：

- URL 是否正确。
- 接口服务是否可访问。
- Header 是否包含接口要求的鉴权信息。
- Body 是否符合接口要求。
- `image_b64` 是否注入到了正确字段。
- 超时时间是否太短。

### 14.4 解析结果为空

请检查：

- `Response` 示例是否和真实响应结构一致。
- `Collection Path` 是否选中了检测结果列表。
- `bbox`、`label`、`class_id`、`conf` 路径是否正确。
- 坐标模式 `BBox/Point` 和 `XYXY/XYWH` 是否与接口实际返回一致。

### 14.5 图片预览没有检测框

请检查：

- 当前图片是否运行成功。
- `解析结果` 是否有 bbox。
- bbox 坐标是否超出图片范围。
- 坐标格式是否选错，例如把 `XYWH` 当作 `XYXY`。

### 14.6 找不到本机文件

`Browse` 显示的是后端机器的文件系统。如果浏览器和后端不在同一台机器上，请使用上传功能把本机文件上传到后端的 `datasets` 目录。

### 14.7 GT 预览失败

请检查：

- 标注格式是否选择正确。
- 标注文件夹是否正确。
- 类别名称顺序是否与标注文件一致。
- 图片文件名和标注文件名是否能对应。

### 14.8 结果评估无法开始

请检查：

- 是否已经至少成功运行过一张图片。
- `解析结果` 中是否有预测框。
- 是否已经填写类别名称。
- 是否已经选择 GT 标注文件夹。
- GT 标注格式是否与文件夹中的标注文件一致。
- 当前 `Image Path` 是否对应参与评估的图片或图片目录。

### 14.9 页面看起来还是旧版本

如果发布后仍然看不到新功能，例如没有 `结果评估` 页签，请先确认浏览器访问的是当前启动程序的地址，例如 `http://127.0.0.1:8000`。确认地址正确后，按 `Ctrl+F5` 强制刷新页面。

如果仍然没有变化，请关闭旧的 `fuxing.exe` 进程后重新启动发布包。
## 15. 使用建议

- 先用单张图片调通，再批量运行。
- 修改 `Body` 或 `Response` 后，重新检查 `响应映射` 和 `请求配置`。
- 保存 Pipeline 后再运行，避免临时配置丢失。
- 响应映射尽量从真实接口响应样例中选择路径，减少手动猜测。
- 批量运行前，先确认图片预览中的检测框位置正确。
- 做结果评估前，先确认 GT 标注格式、类别名称顺序和图片文件名对应关系。
